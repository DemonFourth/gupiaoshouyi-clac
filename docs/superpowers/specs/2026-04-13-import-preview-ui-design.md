# 导入数据预览弹窗 UI 优化设计文档

**日期**：2026-04-13
**版本**：v1.0
**涉及版本**：v2.6.2（计划）

---

## 背景与目标

当前"导入数据预览"弹窗 UI 较为简陋：统计数字以纯文本行罗列，股票明细只有一行文字，用户无法查看具体导入了哪些交易记录、哪些被跳过。

**优化目标**：
1. 视觉更现代，与主应用主色 `#667eea` 风格统一
2. 统计数字更突出，信息层次更清晰
3. 股票明细支持展开查看具体交易记录
4. 操作按钮视觉权重更明确（主操作 vs 危险操作 vs 取消）

---

## 方案选择

**方案 A（已选）**：统计卡片化 + 可展开股票行

- 顶部4个统计数字改为横排卡片
- 股票明细改为可点击展开的行
- 展开后显示具体交易记录表格（日期、类型、价格、数量、状态）
- 底部按钮拉开视觉权重

---

## 详细设计

### 1. 弹窗容器

- `max-width` 从 600px 扩大到 **720px**
- `max-height` 保持 80vh，内容区可滚动

### 2. 统计卡片区（顶部）

横排4张卡片，等宽分布：

| 卡片 | 图标 | 大数字颜色 |
|------|------|-----------|
| 新增股票 | 📦 | `#667eea`（主色，突出） |
| 已存在股票 | 📋 | `#333`（中性） |
| 新增记录 | 📝 | `#667eea`（主色，突出） |
| 重复跳过 | ⏭ | `#999`（弱化） |

每张卡片结构：
```
┌──────────────┐
│  [图标]      │
│  2           │  ← 大数字（20px bold）
│  新增股票    │  ← 标签（12px, #666）
└──────────────┘
```

背景：`#f8f9ff`，左侧4px色条（新增用主色，重复用 `#e0e0e0`），圆角 8px。

### 3. 股票明细区

#### 3.1 分组标题
```
新增股票 (2)
```
字体 13px，颜色 `#666`，margin-bottom 8px。

#### 3.2 股票行（折叠态）
```
┌──────────────────────────────────────────────────────┐
│ ▌ 赣锋锂业 (002460)          15条记录    [▼]        │
└──────────────────────────────────────────────────────┘
```
- 左侧 3px 竖条：新增股票用 `#667eea`，已存在股票用 `#ff9800`
- 股票名称 + 代码：14px bold
- 右侧：记录摘要文字 + 展开箭头按钮
- 已存在股票摘要文字：`新增X条 重复Y条`（若全部重复则显示 `全部重复`）
- hover 背景：`rgba(102,126,234,0.05)`

#### 3.3 股票行（展开态）
展开区域显示该股票的交易记录表格：

| 列 | 内容 | 宽度 |
|----|------|------|
| 日期 | YYYY-MM-DD | 110px |
| 类型 | 买入/卖出/分红/红利税（badge） | 80px |
| 价格 | ¥xxx.xxx | 90px |
| 数量 | xxx股 | 80px |
| 状态 | ✅ 新增 / ⊘ 重复 | 80px |

状态颜色：
- 新增：绿色 `#2e7d32` + ✅
- 重复跳过：灰色 `#999` + ⊘，整行文字也变灰

表格最大高度 200px，超出可滚动。

### 4. 警告区

```
⚠️  注意：覆盖数据将删除当前所有数据，请谨慎操作！
```

- 背景 `#fff3e0`，左边框 4px `#ff9800`
- 图标用 SVG 警告三角，而非 emoji（兼容性更好）
- 字体 13px，颜色 `#e65100`

### 5. 底部按钮

```
[  合并数据  ]   [ 覆盖数据 ]   取消
```

- **合并数据**：`btn-primary`（主色填充，白字）— 最优先操作
- **覆盖数据**：`btn-danger-outline`（红色 border + 红字，无填充）— 危险操作降权
- **取消**：纯文本链接样式（`#666`）— 最低权重

## 数据层变化

### `fileStorage.js` — `analyzeTrades()` 扩展

当前只返回计数，需额外返回具体记录列表供展开渲染：

```javascript
// 当前返回
{ newCount, duplicateCount }

// 新返回
{
  newCount,
  duplicateCount,
  newItems: [...trade],       // 新增的完整 trade 对象列表
  duplicateItems: [...trade]  // 重复的完整 trade 对象列表
}
```

### `fileStorage.js` — `analyzeImportData()` 扩展

将 `newItems` / `duplicateItems` 传递到 `existingStocks` 条目：

```javascript
result.existingStocks.push({
  code, name,
  currentTrades, importTrades,
  newTrades: analysis.newCount,
  duplicateTrades: analysis.duplicateCount,
  newItems: analysis.newItems,       // 新增
  duplicateItems: analysis.duplicateItems  // 新增
});
```

新股票的 `newStocks` 条目也需保存完整 trades 列表用于展开：

```javascript
result.newStocks.push({
  code, name,
  trades: stock.trades.length,
  tradeItems: stock.trades  // 新增
});
```

| 文件 | 改动类型 | 改动内容 |

---

## 不在范围内

- 不引入"冲突"状态（同日期同类型但价格/数量不同仍视为新增）
- 不修改合并/覆盖的业务逻辑
- 不修改数据校验逻辑

---

## 状态说明

| 状态 | 触发条件 | 显示样式 |
|------|---------|---------|
| ✅ 新增 | 四字段（日期-类型-价格-数量）不匹配现有记录 | 绿色文字 |
| ⊘ 重复跳过 | 四字段完全匹配现有记录 | 灰色文字，整行弱化 |
