# 股票收益计算器

**项目定位**：本工具专为**已发生的股票交易操作**提供收益计算和分析功能，针对**持仓中**和**已清仓**的股票进行精确的收益统计。

> ⚠️ **重要说明**：本工具专注于交易记录和收益计算，不提供实时行情推送、价格预警、自选股盯盘等功能。如需盯盘功能，请使用专业的股票交易软件。

## 核心特性

- ✅ **交易记录管理**：记录买入、卖出、分红、红利税等操作
- ✅ **FIFO精确计算**：使用先进先出法计算持仓成本和收益
- ✅ **收益统计分析**：统一口径统计已实现收益和浮动盈亏
- ✅ **持仓周期追踪**：自动识别建仓、加仓、减仓、清仓操作
- ✅ **数据可视化**：通过图表展示收益趋势和成本变化
- ✅ **大数字转换**：自动将大数字转换为万/亿单位，悬浮显示完整数值
- ✅ **云端持久化**：Cloudflare D1 数据库存储，跨设备同步
- ✅ **主题切换**：深色/浅色主题切换，localStorage持久化偏好
- ✅ **备注功能**：交易记录支持添加备注，引用行样式显示
- ✅ **平滑刷新**：编辑后保持滚动位置，无跳动无闪烁
- ✅ **响应式布局**：自适应各种屏幕宽度，自动换行和调整
- ✅ **代码质量保障**：ESLint + stylelint 代码检查，防止未定义变量等问题

## 快速开始

### 1. 打开应用
直接在浏览器中打开 `index.html` 文件即可使用。

**推荐浏览器**：Chrome、Edge、Firefox、Safari

### 2. 添加股票
1. 点击"添加股票"按钮
2. 输入股票代码（6位数字，如：002460）
3. 输入股票名称（如：赣锋锂业）
4. 点击"保存"

**支持的证券类型**：
- 上海股票：6xxxxx（如：600036）
- 深圳股票：0xxxxx、3xxxxx（如：002460）
- 上海ETF：5xxxxx（如：510300）
- 深圳ETF：15xxxx、16xxxx（如：159326）

### 3. 添加交易记录
1. 选择要操作的股票
2. 点击"添加记录"按钮
3. 填写交易表单：
   - 交易日期
   - 操作类型（买入/卖出/分红/红利税补缴）
   - 价格、数量、手续费
4. 点击"确认添加"

### 4. 数据备份与恢复
1. 点击右上角**设置按钮**（齿轮图标）
2. 点击**导出数据**：下载JSON文件备份
3. 点击**导入数据**：选择JSON文件恢复数据

## 数据存储

### 混合存储策略

| 存储层 | 说明 | 特点 |
|--------|------|------|
| **localStorage** | 浏览器本地存储 | 零延迟、即时响应 |
| **Cloudflare D1** | SQLite 边缘数据库 | 云端持久化、跨设备同步 |

### 数据同步流程

**读取**：优先读取 localStorage（零延迟），后台异步检查 D1 数据差异，有差异时弹窗提示。

**写入**：先保存到 localStorage（立即生效），异步同步到 D1（不阻塞 UI）。

### 数据备份

- **导出数据**：设置 → 导出数据 → 下载JSON文件
- **导入数据**：设置 → 导入数据 → 选择JSON文件 → 合并或覆盖

## 收益指标说明

### 核心指标

| 指标 | 计算公式 |
|------|----------|
| **已卖出收益** | Σ(卖出净额 - 对应成本) |
| **当前持仓收益** | 持仓市值 - 持仓成本 |
| **总收益** | 已卖出收益 + 当前持仓收益 |
| **每股摊薄成本** | (持仓市值 - 当前持仓周期收益) ÷ 持仓股数 |
| **总收益率** | (现价 - 每股摊薄成本) ÷ 每股摊薄成本 × 100% |

### 周期收益

```
周期收益 = 周期内卖出收益 + 周期内分红 - 周期内红利税
```

- **周收益**：按自然周（周一到周日）计算
- **月收益**：按自然月计算

### FIFO计算

卖出股票时，优先卖出最早买入的那一批，确保每笔卖出的成本计算准确。部分卖出时，手续费按比例分摊。

## 常见问题

### 本工具是股票盯盘软件吗？
**不是**。本工具专注于交易记录和收益计算，不支持实时行情推送、价格预警、自选股盯盘。

### 数据会丢失吗？
数据存储在浏览器本地，清除浏览器数据会导致丢失。建议定期导出备份。

### 如何在多台电脑间同步数据？
1. 在第一台电脑导出数据
2. 将JSON文件传输到第二台电脑
3. 在第二台电脑导入数据

### 为什么清仓后重新建仓的摊薄成本不一样？
清仓时重置了累计收益，每个持仓周期的摊薄成本计算是独立的。

## 部署说明

### Cloudflare Pages 部署

本项目部署在 Cloudflare Pages 上，使用 D1 数据库进行数据持久化。

#### 部署步骤

1. **创建 D1 数据库**
   ```bash
   npx wrangler d1 create stock-calculator-db
   ```

2. **绑定 D1 数据库**
   - 在 Cloudflare Pages 项目设置中
   - Settings → Functions → D1 database bindings
   - Variable name: `D1`
   - D1 database: `stock-calculator-db`

3. **部署项目**
   ```bash
   # 连接 GitHub 仓库自动部署
   # 或使用 Wrangler 手动部署
   npx wrangler pages deploy
   ```

#### API 端点

| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/data` | GET | 获取所有数据 |
| `/api/data` | PUT | 保存所有数据 |
| `/api/import` | POST | 导入JSON数据 |
| `/api/health` | GET | 健康检查 |

#### 本地开发

本地开发时无需配置 D1 数据库，应用会自动降级使用 localStorage 存储。

```bash
npx live-server .
```
