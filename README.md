# 股票收益计算器

**项目定位**：本工具专为**已发生的股票交易操作**提供收益计算和分析功能，针对**持仓中**和**已清仓**的股票进行精确的收益统计。~~本工具不是股票盯盘或自选股管理工具~~，不支持股票行情监控、价格提醒等盯盘功能。核心功能是帮助个人投资者记录交易历史，使用FIFO算法精确计算成本和收益，并通过图表展示投资趋势。

> ⚠️ **重要说明**：本工具专注于交易记录和收益计算，不提供实时行情推送、价格预警、自选股盯盘等功能。如需盯盘功能，请使用专业的股票交易软件。

**核心特性**：
- ✅ **交易记录管理**：记录已发生的买入、卖出、分红、红利税等操作
- ✅ **FIFO精确计算**：使用先进先出法计算持仓成本和收益
- ✅ **收益统计分析**：统一口径统计已实现收益和浮动盈亏
- ✅ **持仓周期追踪**：自动识别建仓、加仓、减仓、清仓操作
- ✅ **数据可视化**：通过图表展示收益趋势和成本变化
- 📊 **辅助功能**：~~实时股价获取~~（用于计算浮动盈亏，非盯盘功能）

## 功能特性

### 核心功能
- **多股票管理**：支持添加、编辑、删除、切换多只股票
- **股票分组**：持仓中、已清仓两种分组（自动判定）
- **FIFO计算**：先进先出法精确计算成本和收益
- ~~**实时股价**~~：~~自动获取A股和ETF实时价格~~ → **浮动盈亏计算**：获取最新股价用于计算持仓浮动盈亏（非盯盘功能）
- **周期收益统计**：支持本周收益、本月收益统计
- **数据可视化**：年度收益、月度收益、持仓变化、收益变化等趋势图表
- **长时间跨度数据优化**：支持30个月以上数据的图表展示
  - dataZoom缩放功能（滑块 + 鼠标滚轮）
  - 动态标签间隔（根据数据点数量自动调整）
  - X轴日期格式化为月-日，节省空间
- **持仓周期追踪**：自动识别建仓、加仓、减仓、清仓操作
- **加仓对比分析**：显示每次加仓相对于上次加仓的股价变化
- **滚动位置恢复**：从详情页返回汇总页时，恢复到进入详情页时的滚动位置（平滑滚动）

### ~~实时持仓~~ 持仓信息
- ~~当前股价、涨跌幅~~ → 当前股价、涨跌幅（用于计算浮动盈亏，非盯盘功能）
- 持仓成本、持仓市值、持仓股数
- 每股持仓成本、每股摊薄成本
- 当前持仓收益、当前持仓收益率
- 持仓股数悬浮显示每笔买入明细（每笔单独一行显示）

> 💡 **说明**：股价数据仅供参考，用于计算浮动盈亏，不代表实时行情推送或盯盘功能。

### 汇总信息
- 总投入成本、总卖出金额
- 已卖出收益、已卖出收益率
- 总收益（已卖出收益 + 当前持仓收益）
- 总收益率（持仓中按当前持仓周期的摊薄成本计算，清仓股票显示已实现收益率）
- 本周收益、本月收益（统一收益口径：卖出 + 分红 - 红利税）

### 汇总页增强
- **年度收益统计**：按统一收益口径汇总每年收益
- **月度收益趋势**：按统一收益口径汇总选中年份的每月收益
- **Top5榜单**：持仓中/已清仓分别展示盈利TOP5和亏损TOP5
- **名称可读性优化**：Top5列表支持股票全名两行显示，并附带股票代码

### 交易记录管理
- 添加、编辑、删除交易记录
- 支持买入/卖出/分红/红利税补缴四种操作类型
- 自动计算收益金额和收益率
- 收益显示格式：`¥123.45 (1.234%)`
- 持仓周期标记：建仓、加仓、减仓、清仓、分红、红利税

### 持仓周期追踪
- **建仓**：空仓状态下的第一笔买入
- **加仓**：有持仓时的后续买入
- **减仓**：卖出后仍有持仓
- **清仓**：卖出后持仓归零（重置累计收益）
- **分红**：股票分红收入
- **红利税**：红利税补缴
- 自动识别完整持仓周期，标记第X轮

### 持仓明细
- 按FIFO原则显示每笔买入的卖出情况
- 显示卖出日期、价格、数量、收益金额
- 状态标识：持有中、部分卖出、已卖出

### 数据统计
- **统计表格**：买入次数、卖出次数、盈利次数、亏损次数、胜率、平均收益率、最大盈利、最大亏损
- **持仓数量变化趋势**：折线图展示持仓变化
- **累计收益变化趋势**：折线图展示收益变化
- **单笔交易收益**：柱状图展示每笔卖出收益
- **累计收益率变化趋势**：折线图展示收益率变化
- **每股成本趋势图（详情页）**：
  - 每股持仓成本与每股摊薄成本的历史曲线
  - 自动标注"最新价"虚线，便于对比当前价格与成本区间
  - **加仓对比功能**：鼠标悬浮到加仓点时显示加仓对比信息
- **年度收益统计图**：汇总页按统一收益口径展示年度收益
- **月度收益趋势图**：汇总页按统一收益口径展示月度收益

### 数据管理
- **导出数据**：将数据导出为JSON文件，方便备份
- **导入数据**：从JSON文件导入数据，支持预览和合并
- **导入预览**：显示新增股票、已存在股票、新增记录、重复记录
- **二次确认**：导入前预览数据，选择合并或覆盖

## 快速开始

### 1. 打开应用
直接在浏览器中打开 `index.html` 文件即可使用。

**推荐浏览器**：
- Chrome
- Edge
- Firefox
- Safari

### 2. 添加股票
1. 点击"添加股票"按钮
2. 输入股票代码（6位数字，如：002460）
3. 输入股票名称（如：赣锋锂业）
4. 选择分组（持仓中/已清仓）

> 说明：分组会在保存时根据当前持仓股数自动归一化：`currentHolding > 0` 为"持仓中"，否则为"已清仓"。
5. 点击"保存"

**支持的证券类型**：
- 上海股票：6xxxxx（如：600036）
- 深圳股票：0xxxxx、3xxxxx（如：002460）
- 上海ETF：5xxxxx（如：510300、563360）
- 深圳ETF：15xxxx、16xxxx（如：159326）

### 3. 添加交易记录
1. 选择要操作的股票
2. 点击"添加记录"按钮
3. 填写交易表单：
   - 交易日期
   - 操作类型（买入/卖出/分红/红利税补缴）
   - 价格（支持3位小数，如：3.067）
   - 数量
   - 手续费（默认5元）
   - 金额（买入/卖出自动计算，分红/红利税可输入）
4. 点击"确认添加"

**操作类型说明**：
- **买入**：买入股票，需填写价格、数量、手续费，金额自动计算
- **卖出**：卖出股票，需填写价格、数量、手续费，金额自动计算
- **分红**：股票分红收入，只需填写金额
- **红利税补缴**：红利税补缴支出，只需填写金额

### 4. 数据备份与恢复
1. 点击右上角**设置按钮**（齿轮图标）
2. 点击**导出数据**：下载JSON文件备份
3. 点击**导入数据**：选择JSON文件恢复数据
4. 导入时会显示预览，可选择**合并数据**或**覆盖数据**
5. **持仓明细显示开关**：在"显示设置"中控制持仓明细的显示/隐藏

### 5. 持仓明细显示设置
持仓明细（FIFO先进先出）表格显示每笔买入的卖出情况。用户可以通过设置开关控制其显示：
- **显示设置**：在设置弹窗的"显示设置"中切换"持仓明细"开关
- **配置持久化**：开关状态会保存到浏览器配置中，下次访问时自动恢复
- **默认状态**：默认显示持仓明细

### ~~实时持仓~~ 持仓信息
| 项目 | 说明 |
|-----|------|
| ~~当前股价~~ | ~~实时股票价格~~ → 当前股价（用于计算浮动盈亏） |
| 涨跌幅 | 当日涨跌金额和百分比（仅供参考） |
| 持仓成本 | 当前持仓的总成本 |
| 持仓市值 | 当前持仓 × 最新股价 |
| 持仓股数 | 当前持有的股票数量（悬浮显示买入明细） |
| 每股持仓成本 | 持仓成本 ÷ 持仓股数 |
| 每股摊薄成本 | (持仓市值 - 当前持仓周期收益) ÷ 持仓股数 |
| 当前持仓收益 | 持仓市值 - 持仓成本（浮动盈亏） |
| 当前持仓收益率 | 当前持仓收益 ÷ 持仓成本 × 100% |

### 汇总信息
| 项目 | 说明 |
|-----|------|
| 总投入成本 | 所有买入金额 + 手续费 |
| 总卖出金额 | 所有卖出金额 - 手续费 |

## 数据存储

### 存储方式

项目采用 **localStorage + Cloudflare D1 混合存储** 策略，实现本地快速响应和云端数据持久化。

| 存储层 | 说明 | 特点 |
|--------|------|------|
| **localStorage** | 浏览器本地存储 | 零延迟、即时响应 |
| **Cloudflare D1** | SQLite 边缘数据库 | 云端持久化、跨设备同步 |

### 混合存储策略

```
┌─────────────────────────────────────────────────────────────┐
│                    读取流程                                  │
├─────────────────────────────────────────────────────────────┤
│  1. 优先读取 localStorage（零延迟显示）                      │
│  2. 后台异步检查 D1 数据差异                                │
│  3. 有差异时弹窗提示用户选择处理方式                        │
├─────────────────────────────────────────────────────────────┤
│                    写入流程                                  │
├─────────────────────────────────────────────────────────────┤
│  1. 先保存到 localStorage（立即生效）                       │
│  2. 更新内存缓存                                            │
│  3. 异步同步到 D1（不阻塞 UI）                              │
└─────────────────────────────────────────────────────────────┘
```

## FIFO计算说明

### 先进先出法
卖出股票时，优先卖出最早买入的那一批，确保每笔卖出的成本计算准确。

### 计算示例
```
买入记录：
- B1: 2025-10-29 买入 200股 @ 66.09
- B2: 2025-11-18 买入 200股 @ 69.81

卖出记录：
- S1: 2025-12-23 卖出 200股 @ 66.58

FIFO匹配：
- S1 匹配 B1（全部卖出）
- 成本：66.09 × 200 + 5 = 13223元
- 收益：66.58 × 200 - 11.66 - 13223 = 81.34元
- 收益率：0.62%
```

### 手续费分摊
部分卖出时，手续费按比例分摊：
```
B1: 买入 200股，手续费 5元
卖出 100股时，分摊手续费 = 5 × (100/200) = 2.5元
剩余手续费 = 5 - 2.5 = 2.5元
```

| 类型 | 说明 | 标签颜色 |

| 建仓 | 空仓状态下的第一笔买入 | 绿色 |

## 收益指标说明

### 已卖出收益（已实现盈亏）
所有卖出操作的实际盈亏总和，已落袋为安的利润。

**计算公式**：
```
已卖出收益 = Σ(卖出净额 - 对应成本)
```

### 已卖出收益率
已卖出收益占总投入成本的百分比。

**计算公式**：
```
已卖出收益率 = 已卖出收益 ÷ 总投入成本 × 100%
```

### 当前持仓收益（未实现盈亏）
当前持仓按最新股价计算的浮动盈亏，还未实际卖出。

**计算公式**：
```
当前持仓收益 = 持仓市值 - 持仓成本
            = 当前股数 × 最新股价 - 持仓总成本
```

### 当前持仓收益率
当前持仓收益占持仓成本的百分比。

**计算公式**：
```
当前持仓收益率 = 当前持仓收益 ÷ 持仓成本 × 100%
```

### 总收益
已实现收益与浮动收益的总和。

**计算公式**：
```
总收益 = 已卖出收益 + 当前持仓收益
```

### 周收益 / 月收益
使用统一收益口径统计当前自然周、自然月内的收益。

**计算公式**：
```
周期收益 = 周期内卖出收益 + 周期内分红 - 周期内红利税
```

**说明**：
- 周收益：按自然周（周一到周日）计算
- 月收益：按自然月计算
- 汇总页卡片、年度图、月度图使用相同收益口径

### 总收益率（修正版）
总收益率会根据是否仍有持仓采用不同口径。

**计算公式**：
```
每股摊薄成本 = (持仓市值 - 当前持仓周期收益) ÷ 持股数量
总收益率 = (现价 - 每股摊薄成本) ÷ 每股摊薄成本 × 100%
```

**说明**：
- **持仓中股票**：基于当前持仓周期的摊薄成本计算
  - 摊薄成本 = (持仓成本 - 当前持仓周期收益) / 持仓股数
  - **清仓时重置累计收益**，确保每个持仓周期的摊薄成本计算独立
- **已清仓股票**：显示已实现收益率 = 总收益 ÷ 总投入成本 × 100%
- 摊薄成本考虑了当前持仓周期内的已实现收益对持仓成本的影响

### 加仓对比
每次加仓相对于上次加仓的股价变化。

**计算公式**：
```
价格变化 = 本次加仓价格 - 上次加仓价格
变化百分比 = (价格变化 / 上次加仓价格) × 100%
```

**说明**：
- 只在当前持仓周期内有加仓时显示对比
- 清仓后重新建仓时，不会显示"最新价对比最近加仓"
- 鼠标悬浮到每股成本趋势图的加仓点时显示详细信息
- 图表标题显示~~实时股价~~ → 当前股价与最近一次加仓的对比（金额保留3位小数，百分比保留2位小数）
- 图表右侧的最新价标签显示3位小数

## 常见问题

### 1. 本工具是股票盯盘软件吗？
**不是**。本工具专注于交易记录和收益计算，不是股票盯盘或自选股管理工具。
- ✅ 支持：记录交易历史、计算收益、统计数据分析
- ❌ 不支持：实时行情推送、价格预警、自选股盯盘

如需盯盘功能，请使用专业的股票交易软件（如同花顺、东方财富等）。

### 2. 数据会丢失吗？
数据存储在浏览器本地，以下情况会导致数据丢失：
- 清除浏览器数据
- 使用隐私模式浏览
- 更换浏览器或电脑

**建议**：定期导出数据备份。

### 3. 如何在多台电脑间同步数据？
1. 在第一台电脑上导出数据
2. 将JSON文件传输到第二台电脑
3. 在第二台电脑上导入数据

### 4. 导入数据时选择合并还是覆盖？
- **合并数据**：保留现有数据，添加新数据，重复记录自动跳过
- **覆盖数据**：删除现有数据，使用导入的数据替换

### 5. 股价获取失败怎么办？
- 检查网络连接
- 股票API可能有访问限制
- 可以手动输入股价进行计算

### 6. ETF价格无法输入？
- 已支持3位小数输入
- 价格输入框step="0.001"
- 显示时智能判断：第3位为0显示2位，否则显示3位

### 7. 为什么清仓后重新建仓的摊薄成本不一样？
- 这是正常现象，因为清仓时重置了累计收益
- 每个持仓周期的摊薄成本计算是独立的
- 这样可以准确反映当前周期的投资回报率

### 数据流
```
用户操作 → TradeManager → DataManager → localStorage
                ↓
           Calculator → App.updateAll() → UI更新
```

**性能优化数据流**（v2.2.0）：
```
用户操作 → TradeManager → DataService.getCalculationResult()
                ↓                      ↓
           Calculator        检查缓存 → 命中：返回缓存结果
                ↓                      ↓
           计算结果        未命中：计算后缓存并返回
                ↓
           StockSnapshot  →  快照缓存失效 → 重新计算
```

### 性能优化最佳实践

1. **DOM缓存优化**（性能提升 50-80%）：
   - 所有模块都必须实现 DOM 缓存
   - 提供 `initDOMCache()` 方法初始化缓存
   - 提供 `_ensureDOMCache()` 方法确保缓存已初始化
   - 在模块初始化时调用 `initDOMCache()`
   - 缓存所有需要频繁访问的 DOM 元素

2. **批量DOM插入优化**（性能提升 60-90%）：
   - 插入多个DOM元素时，必须使用 DocumentFragment
   - 禁止逐个插入DOM元素（除非有特殊原因）
   - 使用 `document.createDocumentFragment()` 创建文档片段
   - 在片段中批量添加 DOM 元素
   - 最后一次性插入到页面中

3. **懒加载优化**（性能提升 40-60%）：
   - 图表等重型组件必须使用 IntersectionObserver 实现懒加载
   - 设置合理的 threshold（推荐 0.1）
   - 设置合理的 rootMargin（推荐 100px）
   - 图表只在进入视口时才渲染
   - 使用 `IntersectionObserver` API 监听图表容器

4. **防抖和节流**：
   - 频繁触发的事件（如 resize、scroll）必须使用防抖或节流
   - 推荐使用防抖，默认延迟 200ms
   - 必须在模块销毁时清理定时器
   - 使用 `setTimeout` 和 `clearTimeout` 实现防抖
   - 统一管理所有防抖处理函数

5. **数据缓存优化**（性能提升 70-90%）：
   - 计算结果必须缓存，避免重复计算
   - 提供缓存失效机制
   - 数据变更时自动失效相关缓存
   - 使用 Map 存储计算结果缓存
   - 通过 EventBus 监听数据变更事件，自动失效缓存

6. **事件监听器清理**：
   - 所有事件监听器必须在适当的时候解绑
   - 提供清理方法，在模块销毁时调用
   - 防止内存泄漏
   - 保存事件处理函数引用，以便取消订阅
   - 模块销毁时清理所有事件监听器

3. 在 `app.js` 中初始化新模块
