# 股票收益计算器

> 基于 FIFO 算法的股票投资收益计算工具

## 项目简介

**股票收益计算器**是一款专为个人投资者设计的前端 Web 应用，用于记录已发生的股票交易操作并精确计算收益。本工具针对**持仓中**和**已清仓**的股票进行收益统计，帮助投资者清晰掌握投资情况。

> **重要说明**：本工具专注于交易记录和收益计算，**不是**股票盯盘软件或自选股管理工具，不支持实时行情推送、价格预警等功能。

### 核心特性

- **交易记录管理** - 支持买入、卖出、分红、红利税四种操作类型
- **FIFO 精确计算** - 使用先进先出法计算持仓成本和收益
- **收益统计分析** - 统一口径统计已实现收益和浮动盈亏
- **持仓周期追踪** - 自动识别建仓、加仓、减仓、清仓操作
- **数据可视化** - 多维度图表展示收益趋势和成本变化
- **混合存储策略** - 本地优先 + 云端同步，零延迟体验
- **一键导入导出** - 支持 JSON 和 CSV 格式数据备份

## 功能特性

### 交易记录管理

| 操作类型 | 说明 |
|----------|------|
| 买入 | 记录买入价格、数量、手续费 |
| 卖出 | 记录卖出价格、数量、手续费，自动计算收益 |
| 分红 | 记录股票分红收入 |
| 红利税 | 记录红利税补缴 |

### FIFO 计算算法

```
卖出成本计算流程：
1. 从最早的买入批次开始匹配
2. 完全卖出该批次 → 计算完整成本
3. 部分卖出 → 按比例分摊手续费
4. 继续匹配下一批次，直到卖出数量完成
```

**优势**：
- 精确计算每笔卖出的实际成本
- 支持部分卖出场景
- 正确处理手续费分摊

### 收益指标

| 指标 | 计算方式 |
|------|----------|
| 已卖出收益 | 所有卖出操作的实际盈亏总和 |
| 当前持仓收益 | 持仓市值 - 持仓成本 |
| 总收益 | 已卖出收益 + 当前持仓收益 |
| 总收益率 | (现价 - 每股摊薄成本) / 每股摊薄成本 × 100% |
| 周期收益 | 卖出 + 分红 - 红利税 |
| 每股摊薄成本 | (持仓成本 - 当前持仓周期收益) / 持仓股数 |

### 数据可视化

**汇总页图表**：
- 年度收益统计图
- 月度收益趋势图
- Top5 盈利/亏损榜单

**详情页图表**：
- 持仓数量变化趋势
- 累计收益变化趋势
- 单笔交易收益柱状图
- 累计收益率变化趋势
- 每股成本趋势（含加仓对比）

**交易记录页图表**：
- 股票交易额排行（气泡图/柱状图等）
- 每日交易额统计
- 每日收益趋势
- 交易类型分布（饼图/环形图）
- 股票收益排行
- 每日交易次数

**图表特性**：
- dataZoom 缩放功能（滑块 + 鼠标滚轮）
- 动态标签间隔
- 支持 30 个月以上数据展示

## 存储方案

### 方案对比

| 特性 | 本地模式 (file://) | 部署模式 (Cloudflare Pages) |
|------|-------------------|---------------------------|
| 存储位置 | 浏览器 localStorage | localStorage + D1 数据库 |
| 读取延迟 | 零延迟 | 零延迟（本地优先） |
| 云端同步 | 不支持 | 支持 |
| 多设备访问 | 不支持 | 支持 |
| 数据备份 | 手动导出 | 自动云端备份 |

### 混合存储策略

**读取流程**：
```
1. 优先读取 localStorage（零延迟显示）
2. 后台异步检查 D1 数据差异
3. 有差异时弹窗提示用户选择处理方式
```

**写入流程**：
```
1. 先保存到 localStorage（立即生效）
2. 更新内存缓存
3. 异步同步到 D1（不阻塞 UI）
```

### API 端点

| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/data` | GET | 获取所有数据 |
| `/api/data` | PUT | 保存所有数据 |
| `/api/import` | POST | 导入 JSON 数据 |
| `/api/health` | GET | 健康检查 |

### D1 数据库结构

```sql
CREATE TABLE app_data (
    key VARCHAR(50) PRIMARY KEY,
    value TEXT NOT NULL,
    last_updated TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## 部署指南

### 前置条件

- GitHub 账号
- Cloudflare 账号

### 部署步骤

1. **Fork 或 Clone 本仓库**

2. **创建 D1 数据库**
   ```bash
   npx wrangler d1 create stock-calculator-db
   ```

3. **在 Cloudflare Pages 创建项目**
   - 登录 Cloudflare Dashboard
   - 进入 Pages → 创建项目
   - 连接 GitHub 仓库
   - 选择 `gupiaoshouyi-clac` 仓库

4. **绑定 D1 数据库**
   - 进入项目设置 → Functions
   - 添加 D1 数据库绑定
   - Variable name: `D1`
   - 选择创建的数据库

5. **部署完成**
   - 自动触发部署
   - 访问分配的域名

### 本地开发配置

如需本地测试 Functions：

```bash
# 安装 wrangler
npm install -g wrangler

# 本地运行
npx wrangler pages dev .
```

## 使用教程

### 添加股票

1. 点击页面顶部"添加股票"按钮
2. 输入股票代码（6位数字）和股票名称
3. 选择分组（持仓中/已清仓）
4. 点击确认保存

### 添加交易记录

1. 在股票列表中选择目标股票
2. 点击"添加记录"按钮
3. 填写交易信息：
   - 交易日期
   - 交易类型（买入/卖出/分红/红利税）
   - 价格（买入/卖出必填）
   - 数量（买入/卖出必填）
   - 手续费（可选）
4. 点击确认保存

### 查看收益

- **汇总页**：查看所有股票的整体收益、年度/月度收益统计
- **详情页**：查看单只股票的详细收益、持仓明细、交易记录
- **交易记录页**：按时间段筛选查看交易记录和多维度图表

### 数据备份

1. 点击右上角设置按钮（齿轮图标）
2. 选择导出方式：
   - 导出 JSON：完整数据备份
   - 导出 CSV：详细数据或汇总数据
3. 选择保存位置

### 数据恢复

1. 点击右上角设置按钮
2. 点击"导入数据"
3. 选择 JSON 文件
4. 预览导入内容
5. 选择合并或覆盖现有数据

## 注意事项

### 数据安全

- **本地模式**：数据保存在浏览器 localStorage，清除浏览器数据会导致数据丢失
- **部署模式**：数据同步到云端，但仍建议定期导出备份
- **跨设备**：本地模式不支持跨设备同步

### 浏览器兼容性

| 浏览器 | 最低版本 |
|--------|----------|
| Chrome | 90+ |
| Edge | 90+ |
| Firefox | 88+ |
| Safari | 14+ |

### 已知限制

1. 股价数据来自腾讯股票 API，可能有访问限制
2. 金额计算使用浮点数，显示时使用 `toFixed()` 格式化
3. 日期使用 YYYY-MM-DD 格式，避免时区问题
